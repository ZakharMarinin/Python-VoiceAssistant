# webbrowser

Библиотека `webbrowser` - это библиотека, которая позволяет нам управлять браузером на компьютере.

Она встроена в Python по умолчанию, поэтому её не нужно устанавливать через `pip`.

[Документация к библиотеке](https://docs.python.org/3/library/webbrowser.html)

### 1. Как она работает (Принцип "Пульта управления")

Библиотека не знает, какой именно браузер у вас установлен (Chrome, Firefox, Brave или какой-то редкий браузер в Linux). Она работает как **пульт управления операционной системой**.

Когда вы вызываете функцию `webbrowser.open()`, Python делает следующее:

1. Обращается к операционной системе (Windows, Linux или macOS).
    
2. Спрашивает: _"Привет, ОС! Какой браузер у пользователя стоит по умолчанию?"_.
    
3. Получает ответ и дает команду системе: _"Эй, открой эту ссылку вот в этом браузере!"_
### 2. Ключевые функции

В вашем коде вы использовали `webbrowser.open(url, new=2)`, давайте разберем, что это значит:

- **`webbrowser.open(url)`**: Просто открывает страницу. Если вкладка уже открыта - может просто переключиться на неё.
    
- **Параметр `new`**:
    
    - `new=0`: Открывает ссылку в том же окне/вкладке (если это возможно).
        
    - `new=1`: Принудительно открывает в **новом окне**.
        
    - `new=2`: Принудительно открывает в **новой вкладке** (именно это вы используете для YouTube и поиска Google, чтобы не закрывать ассистента).
### 3. Зачем она нужна в вашем проекте?

Ваш голосовой ассистент не "видит" YouTube и не умеет искать в Google внутри себя. Он лишь **генерирует правильный адрес (URL)**.

Посмотрите на вашу функцию `Youtube`:

Python

```
def youtube_search(text):
    tes = urllib.parse.quote_plus(text) # Превращает "kitty videos" в "kitty+videos"
    url = 'https://www.youtube.com/results?search_query='
    webbrowser.open(url + tes, new=2)
```

Это классический пример работы с **Query Parameters (Параметрами запроса)**:

1. У YouTube есть «шаблон» поиска: `youtube.com/results?search_query=...`
    
2. Мы подставляем туда свой запрос после знака `=`.
    
3. Библиотека `urllib.parse.quote_plus` делает запрос "безопасным" для интернета (заменяет пробелы на плюсики, спецсимволы на коды).
    
4. `webbrowser` "выталкивает" этот готовый адрес в ваш браузер.
### 4. Ограничения (Важно для понимания)

- **Отсутствие контроля**: Как только `webbrowser` открыл страницу, он "забывает" о ней. Вы не можете через эту библиотеку прокрутить страницу, нажать кнопку "Play" или закрыть вкладку. Это "одноразовая" команда.
    
- **Зависимость от ОС**: Если в системе пользователя не настроен браузер по умолчанию, библиотека просто выдаст ошибку или не сработает.
