# SpeechRecognition

Библиотека `SpeechRecognition` - это библиотека, позволяющая переводить речь в текст. Она сама не занимается сложной математикой распознавания звука (нейросетями), зато она умеет общаться со всеми популярными API распознавания речи в мире.

[Документация к библиотеке](https://pypi.org/project/SpeechRecognition/)

### 1. Установка

Для того чтобы установить данную библиотеку, выполните в терминале следующую команду:

```
pip install SpeechRecognition
```

### 2. Как она устроена внутри (Архитектура)

Библиотека состоит из трех главных уровней:

1. **Уровень захвата (Microphone):** Работает как интерфейс к вашей звуковой карте. Она не записывает звук сама, а использует внешние аудио-библиотеки (например, `PyAudio`), чтобы "слушать" поток данных.
    
2. **Уровень обработки (AudioData):** Данные с микрофона приходят в виде сырого цифрового шума. Библиотека превращает их в объект `AudioData`, который она умеет правильно упаковывать для отправки на сервер (в форматы WAV или FLAC).
    
3. **Уровень API-шлюза (Recognizers):** Это "мозги" библиотеки. У `SpeechRecognition` нет своего локального "ядра" для понимания слов. Вместо этого у неё есть набор готовых функций для обращения к облачным сервисам: Google Speech Recognition, IBM Speech to Text, Microsoft Bing Speech и др.
### 3. Как работает процесс распознавания

Весь процесс можно разбить на четыре логических этапа:

1. **Калибровка (`adjust_for_ambient_noise`):** Библиотека делает "снимок" тишины. Она анализирует фоновый шум (гул кулера, шум улицы), чтобы потом вычесть его из вашего голоса. Это делает распознавание в разы точнее.
    
2. **Запись и ограничение:** Когда вы вызываете `listen()`, библиотека начинает "слушать". Она использует **Voice Activity Detection (VAD)** — примитивный алгоритм, который понимает, что вы начали говорить, и перестает слушать, как только наступает пауза.
    
3. **Упаковка:** Сырые аудио-данные сжимаются и конвертируются в формат, который ожидает сервер. Например, для Google Speech API библиотека отправляет запрос по протоколу HTTPS.
    
4. **Разбор ответа:** Сервер возвращает ответ в формате JSON. Библиотека парсит этот ответ, достает оттуда самый вероятный текст и выдает его вам в переменную (строку).
### 4. Почему она работает "в облаке"?

Это самый важный момент для ваших учеников. Внутри библиотеки `recognize_google()` спрятан стандартный HTTP-запрос.

- Когда вы говорите что-то в микрофон, ваша фраза улетает на сервер Google.
    
- Там работают огромные нейросети, которые весят терабайты.
    
- Они превращают звук в текст и присылают его обратно.

**Именно поэтому:**

- Если пропадет интернет — программа выдаст ошибку `RequestError`.
    
- Если у Google изменится протокол — библиотека может перестать работать (поэтому разработчики постоянно её обновляют).
