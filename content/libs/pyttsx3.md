# pyttsx3

Библиотека `pyttsx3` - это одна из самых популярных библиотек для **TTS** (Text-to-Speech, преобразование текста в речь) на Python. Её главная особенность - **полная автономность (оффлайн)**. В отличие от Google TTS или нейросетевых облачных сервисов, ей не нужен интернет, так как она использует «движки» синтеза речи, уже установленные в вашей операционной системе.

[Документация к библиотеке](https://github.com/nateshmbhat/pyttsx3)

### 1. Установка

Для того чтобы установить данную библиотеку, выполните в терминале следующую команду:

```
pip install pyttsx3
```

### 2. Архитектура: "Движок" внутри системы

Библиотека `pyttsx3` - это не самостоятельный генератор голоса, а **обертка (интерфейс)**. Она выступает посредником между вашим кодом и системными драйверами озвучки:

- **На Windows:** Она подключается к **SAPI5** (Speech API версии 5), который встроен в Windows по умолчанию.
    
- **На Linux:** Она взаимодействует с **eSpeak NG**. Это мощный и легкий инструмент, который "читает" текст, генерируя звуковые волны.
    
- **На macOS:** Она использует **NSSpeechSynthesizer**, встроенный в систему Apple.
### 3. Основные этапы работы (Цикл событий)

Работа `pyttsx3` строится на концепции **очереди сообщений** и **цикла событий (Event Loop)**.

1. **Инициализация (`init`)**: Вы вызываете `pyttsx3.init()`. В этот момент библиотека находит "родной" движок вашей ОС и подготавливает его к работе.
    
2. **Постановка в очередь (`say`)**: Когда вы вызываете `engine.say("Hello")`, текст не превращается в звук мгновенно. Он попадает во внутреннюю очередь (как в принтер). Это позволяет выстроить несколько фраз в ряд.
    
3. **Запуск цикла (`runAndWait`)**: Это самая важная команда. Она запускает "двигатель", который начинает обрабатывать очередь.
    
    - Программа **блокируется** (останавливается) на этой строке.
        
    - Библиотека по очереди берет текст из очереди, отправляет его в системный драйвер, получает аудио-поток и передает его на аудио-выход (колонки).
        
    - Как только очередь пуста, `runAndWait()` завершается, и управление возвращается вашему коду.
### 4. Как устроены "Голоса"

Каждый системный движок предоставляет список доступных голосов.

- Каждый голос — это объект, у которого есть `id` (уникальный идентификатор), имя, язык и пол.
    
- Когда мы делаем `setProperty('voice', voice.id)`, мы по сути говорим системному драйверу: _"Начиная с этого момента, используй именно этот набор параметров частоты и тембра"_.
