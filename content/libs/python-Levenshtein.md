# python-Levenshtein

Библиотека `python-Levenshtein` — это высокопроизводительный модуль, написанный на языке C, который предназначен для быстрого вычисления метрик схожести строк. В то время как многие высокоуровневые библиотеки используют текстовые вычисления внутри кода на Python, данный модуль работает напрямую с памятью и структурами C, что делает его незаменимым при обработке больших объемов текстовых данных.

Она используется как самостоятельный инструмент для поиска опечаток, так и в качестве мощного «движка» (акселератора) для других библиотек, таких как `fuzzywuzzy` или `RapidFuzz`.

[Документация к библиотеке](https://pypi.org/project/python-Levenshtein/)

### 1. Установка

Для того чтобы установить данную библиотеку, выполните в терминале следующую команду:

```
pip install python-Levenshtein

```

### 2. Как это работает (Ускорение вычислений)

В отличие от стандартных функций Python, которые интерпретируются построчно, `python-Levenshtein` компилируется в нативный машинный код.

* Когда вы вызываете расчет расстояния редакторских правок (вставок, удалений, замен), данные передаются из Python вглубь C-расширения.
* Вычисления происходят на уровне процессора без накладных расходов виртуальной машины Python.

Это позволяет выполнять миллионы сравнений строк в секунду, превращая медленный алгоритмический поиск в мгновенную операцию.

### 3. Главные инструменты (Методы)

Библиотека предоставляет прямой доступ к математическим функциям сравнения строк. Вот основные из них:

* **`Levenshtein.distance(str1, str2)` (Расстояние Левенштейна)**: Возвращает чистое количество правок (целое число), необходимых для приведения одной строки к другой.
* *Пример*: `Levenshtein.distance("cat", "cats")` вернет `1`.


* **`Levenshtein.ratio(str1, str2)` (Коэффициент сходства)**: Возвращает уровень похожести строк в виде числа от 0.0 до 1.0, рассчитываемый на основе длины строк и расстояния Левенштейна.
* *Пример*: `Levenshtein.ratio("Привет", "Приветик")` даст результат около `0.85`.


* **`Levenshtein.jaro_winkler(str1, str2)` (Метрика Яро-Винклера)**: Другой алгоритм сравнения, который дает больший вес строкам, имеющим одинаковое начало (префикс). Идеально подходит для поиска совпадений в именах и фамилиях.
* *Пример*: Отлично находит схожесть в словах с одинаковым корнем, но разными окончаниями.



### 4. Как бы это работало без python-Levenshtein

Без установленной библиотеки `python-Levenshtein` другие инструменты поиска нечетких совпадений (например, `fuzzywuzzy`) переключаются на стандартный модуль Python `difflib`.

Если ваш голосовой ассистент обрабатывает одну фразу в секунду, разница незаметна. Но если бы вы захотели сделать поиск команды по огромной локальной базе данных (например, искать песню среди 10 000 треков), ваш код бы "задумался" на несколько секунд:

Python

```python
# Без python-Levenshtein: чистый Python считает долго
# Программа зависает при переборе большого списка триггеров
for command in huge_database:
    if pure_python_ratio(user_input, command) > 0.8:
        # Ждем ответа...

```

С `python-Levenshtein` этот перебор происходит за миллисекунды. Ассистент мгновенно понимает контекст, даже если база возможных фраз и команд вырастет в сотни раз.

### 5. Особенности (Связь с fuzzywuzzy)

Вам не нужно вручную вызывать методы `Levenshtein.distance` в коде вашего ассистента, если вы уже настроили логику через `fuzzywuzzy`. Библиотека `fuzzywuzzy` спроектирована по принципу модульности: при запуске она проверяет окружение вашей операционной системы. Если она находит установленный модуль `python-Levenshtein`, она автоматически подменяет свои медленные функции на быстрые C-функции «под капотом». Это обеспечивает вашему приложению максимальную скорость работы при сохранении простого синтаксиса.
