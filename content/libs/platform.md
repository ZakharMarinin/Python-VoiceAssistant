# platform

Библиотека `platform`. Она позволяет программе узнать, где именно она запущена: на мощном сервере с Linux, домашнем ноутбуке с Windows или на MacBook.

Она входит в стандартную библиотеку Python, поэтому её не нужно устанавливать отдельно.

[Документация к библиотеке](https://docs.python.org/3/library/platform.html)

### 1. Как она устроена

Библиотека `platform` опрашивает ядро операционной системы и возвращает информацию в виде строк. Она не делает никаких догадок - она просто запрашивает системную информацию напрямую у ОС.
### 2. Главные инструменты в вашем проекте

В вашем коде вы используете одну самую важную функцию - `platform.system()`.

- **`platform.system()`**: Возвращает название ОС.
    
    - На Windows вернет `"Windows"`.
        
    - На Linux вернет `"Linux"`.
        
    - На macOS вернет `"Darwin"`.

Это позволяет вам писать **ветвление (if/else)**, которое делает код универсальным:

Python

```
if platform.system() == "Windows":
    os.system("start mspaint") # Команда Windows
elif platform.system() == "Linux":
    subprocess.Popen(["pinta"]) # Команда Linux
```
### 3. Другие полезные возможности (для эрудиции)

Помимо имени системы, `platform` умеет рассказывать и другие детали, которые могут пригодиться ученикам для создания более продвинутых ассистентов:

- **`platform.version()`**: Возвращает версию системы (например, номер сборки Windows 10 или версию ядра Linux).
    
- **`platform.machine()`**: Показывает архитектуру процессора (например, `AMD64` или `arm64`). Это важно, если вы пишете программу, которая должна по-разному работать на обычных ПК и на Raspberry Pi (которые работают на ARM-процессорах).
    
- **`platform.python_version()`**: Показывает версию Python, на которой сейчас работает скрипт. Это полезно для проверки совместимости (например, если функция работает только в Python 3.10+).

### 4. Почему без неё ваш проект был бы "однобоким"

Без этой библиотеки ваш ассистент был бы "привязан" только к одной ОС.

Представьте ситуацию: вы написали команду `os.system("start mspaint")`.

- **На Windows**: Всё сработает отлично.
    
- **На Linux**: Система выдаст ошибку `command not found: start`, потому что в Linux нет такой команды.

`platform` - это способ сказать программе: _"Посмотри вокруг, где ты находишься, и используй только те инструменты, которые разрешены в этом доме"_.
