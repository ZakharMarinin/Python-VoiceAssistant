# urllib.parse

Библиотека `urllib.parse` - это библиотека, которая позволяет разбирать URL (ссылки), разбирать их на части, а так же за правильностью написания запроса.

В вебе нельзя просто отправить ссылку с пробелами, русскими буквами или спецсимволами (например, `?`, `&`, `#`), так как они имеют служебное значение. `urllib.parse` приводит всё к «безопасному» виду.

Она входит в стандартную библиотеку Python, поэтому её не нужно устанавливать отдельно.

[Документация к библиотеке](https://docs.python.org/3/library/urllib.parse.html)

### 1. Зачем она нужна в вашем проекте?

Ваш ассистент ищет информацию в Google и YouTube. Пользователь может сказать: `"find me kitty videos"`. Если вы просто подставите это в URL, получится: `google.com/search?q=find me kitty videos` - браузер «сломается» на пробелах, так как пробел в URL означает конец адреса.

Функция `quote_plus`, которую вы используете, превращает эту строку в `find+me+kitty+videos`. Теперь это валидный адрес, который поймет любой сервер.
### 2. Главные инструменты `urllib.parse`

#### `quote_plus(string)` - ваш главный инструмент

Она кодирует строку так, чтобы её можно было безопасно использовать в URL.

- **Пробелы** превращаются в `+`.
    
- **Спецсимволы** (например, `/`, `&`, `?`) превращаются в коды вида `%2F`, `%26`.
    
- **Кириллица** превращается в набор байтовых кодов (например, `Привет` станет `%D0%9F%D1%80%D0%B8...`).
#### `urlparse(url)` - «разборщик» адресов

Если вам нужно не создать, а **прочитать** ссылку, эта функция разбивает длинный URL на части:

Python

```
from urllib.parse import urlparse
o = urlparse('https://www.youtube.com/results?search_query=cat')
print(o.netloc) # Выведет: www.youtube.com
print(o.path)   # Выведет: /results
print(o.query)  # Выведет: search_query=cat
```

#### `urlencode(dict)` - для сложных запросов

Часто запросы состоят из нескольких параметров, например: `search?q=cat&lang=ru`. Вместо того чтобы собирать эту строку вручную, вы даете библиотеке словарь:

Python

```
from urllib.parse import urlencode
params = {'q': 'cat', 'lang': 'ru'}
print(urlencode(params)) # Выведет: q=cat&lang=ru
```
### 3. Почему без неё нельзя?

Без `urllib.parse` ваш ассистент был бы крайне ограничен. Представьте, что вы хотите найти в Google запрос `C++ vs Python`.

- Без обработки: `...search?q=C++ vs Python` (браузер подумает, что `vs Python` - это лишние параметры).
    
- С `urllib.parse`: `...search?q=C%2B%2B+vs+Python` (браузер четко видит, что `q` - это весь запрос целиком).
