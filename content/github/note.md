# Руководство по работе с Git и GitHub

### 1. Основные понятия

* **Git** — распределенная система управления версиями, которая устанавливается на локальный компьютер. Она отслеживает изменения в файлах, фиксирует историю состояний проекта и позволяет возвращаться к предыдущим точкам сохранения.
* **GitHub** — веб-платформа для хостинга репозиториев и совместной разработки. Она служит удаленным хранилищем для локальных репозиториев Git, предоставляя инструменты для рецензирования кода, управления задачами и автоматизации сборки (CI/CD).

### 2. Терминология

* **Репозиторий (Repository)** — каталог проекта, содержащий файлы, историю изменений и метаданные Git. Существует в локальном (на ПК) и удаленном (на сервере) виде.
* **Коммит (Commit)** — зафиксированное состояние репозитория. Содержит снимок измененных файлов, хэш-идентификатор, автора, дату и сопроводительное сообщение.
* **Индекс (Staging Area)** — промежуточная область, куда добавляются изменения перед фиксацией в коммит.
* **Ветка (Branch)** — независимая линия разработки. Основная ветка по умолчанию обычно называется `main` или `master`.
* **HEAD** — указатель на текущий активный коммит или ветку в локальном репозитории.
* **Push (Отправка)** — процесс передачи локальных коммитов в удаленный репозиторий.
* **Pull (Получение/Слияние)** — процесс загрузки изменений из удаленного репозитория и их автоматического объединения с текущей локальной веткой.

---

### 3. Схема рабочего процесса (Workflow)

Документ провисает через три основных логических зоны локальной машины перед отправкой на сервер:

1. **Рабочий каталог (Working Directory):** Текущее состояние файлов на диске, где вносятся изменения.
2. **Индекс (Staging Area):** Область подготовки файлов с помощью команды `git add`.
3. **Локальный репозиторий (.git):** База данных проекта, где хранятся коммиты после `git commit`.
4. **Удаленный репозиторий (Remote):** Финальная точка синхронизации данных на GitHub после `git push`.

### 4. Основные консольные команды

#### Инициализация и клонирование

* Создание нового локального репозитория в текущей папке:
```bash
git init
```


* Клонирование существующего удаленного репозитория на локальный диск:
```bash
git clone <url_репозитория>
```

#### Ежедневный цикл разработки

* Просмотр статуса рабочего каталога и индекса (список измененных, удаленных и неотслеживаемых файлов):
```bash
git status
```

* Добавление изменений в индекс (подготовка к коммиту):
```bash
git add .
```

*(Точка указывает на добавление всех измененных файлов в текущем каталоге).*

* Фиксация изменений из индекса в локальный репозиторий с добавлением описания:
```bash
git commit -m "Сообщение коммита"
```

* Отправка локальных коммитов в ветку `main` удаленного репозитория `origin`:
```bash
git push origin main
```

#### Синхронизация и просмотр истории

* Получение актуальной версии кода из удаленного репозитория:
```bash
git pull origin main
```

* Просмотр истории коммитов текущей ветки:
```bash
git log --oneline
```

*(Флаг `--oneline` выводит каждый коммит в одну строку. Выход из интерфейса просмотра — клавиша `Q`).*

### 5. Распространенные ошибки и их устранение

#### Ошибка слияния при отправке (Push Rejected)

```text
[rejected] main -> main (fetch first)
error: failed to push some refs to '...'
```

* **Причина:** Удаленный репозиторий содержит коммиты, которые отсутствуют в локальной копии.
* **Решение:** Выполнить `git pull origin main` для загрузки и слияния удаленных изменений, устранить конфликты (если они возникнут), после чего повторить `git push`.
https://git-scm.com
#### Отмена незакоммиченных изменений

* Сброс изменений в конкретном файле до состояния последнего коммита:
```bash
git checkout -- <имя_файла>
```
