# Установка зависимостей

### Общие библиотеки (Устанавливают все)

```
pip install SpeechRecognition pyttsx3 fuzzywuzzy python-Levenshtein num2words PyAudio
```

**За что отвечает каждая библиотека:**

- `SpeechRecognition` - "уши" программы, захватывает звук и отправляет на распознавание.
    
- `pyttsx3` - "голос" программы, преобразует текст в речь в режиме оффлайн.
    
- `fuzzywuzzy` и `python-Levenshtein` - алгоритмы нечеткого поиска, чтобы ассистент понимал команды, даже если ошибся в одной букве.
    
- `num2words` - переводчик чисел в текст (чтобы синтезатор правильно читал время).
    
- `PyAudio` - системный мостик, который разрешает Python подключаться напрямую к микрофону.

### Установка на Windows

На Windows встроенный синтезатор речи (SAPI5) работает сразу, поэтому `pyttsx3` проблем не вызовет. Главное правило для успешной установки `PyAudio` без танцев с бубном и компиляторами C++:

**Следует использовать стабильную версию Python: 3.12 или 3.13.** В этих версиях библиотека `PyAudio` устанавливается автоматически готовым файлом. Если у кого-то установлена бета-версия (например, 3.14), им нужно:

1. Либо установить Python 3.13 и запустить команду установки через него: `py -3.13 -m pip install pyaudio`
    
2. Либо использовать виртуальное окружение на базе стабильной версии.

### Установка на Linux

На Linux обычного `pip install` недостаточно, так как системе требуются аудио-движки на уровне ОС. Перед тем как устанавливать Python-библиотеки, вам нужно выполнить одну команду в системном терминале:

```
sudo dnf install espeak-ng portaudio-devel python3-devel redhat-rpm-config
```

**Для чего это нужно:**

- `espeak-ng` - это сам движок синтезатора речи для Linux. Без него `pyttsx3` просто не сможет говорить.
    
- `portaudio-devel` и `python3-devel` - это заголовочные файлы, которые нужны пакетному менеджеру `pip`, чтобы успешно собрать и установить мостик к микрофону (`PyAudio`).

# Объяснение кода

Далее будет описана структура работы нашего кода. Весь код будем писать на занятии. По мере продвижения буду обновлять код на старнице для тех, кто пропустил/не успел или просто решил сравнить варинаты.
## Разбор архитектуры: Как это работает?

Код построен на классическом цикле **«Услышал → Очистил → Распознал → Выполнил»**.

### Блок 1: Системный "глушитель" (CTypes)

В Linux звуковая подсистема ALSA выдает много технического спама при опросе портов. Этот блок перехватывает ошибки на уровне языка C и прячет их, чтобы консоль оставалась чистой. На Windows он безопасно игнорируется благодаря `except OSError`.

### Блок 2 и 3: Голос и Слух (`speak` и `command`)

- **`speak()`**: Использует оффлайн-движок `pyttsx3`. Важный нюанс: движок создается (`engine = pyttsx3.init()`) заново при каждом вызове функции. Это защищает программу от известного бага с зависанием очереди озвучки в цикле.
    
- **`command()`**: Записывает 1 секунду тишины для анализа фонового шума (`adjust_for_ambient_noise`), затем слушает пользователя и отправляет запись на сервер Google. При ошибках сети или неразборчивой речи функция вызывает саму себя (рекурсия) и продолжает слушать.
    

### Блок 4 и 5: Словари и Нечеткий поиск

Люди говорят неструктурированно. Библиотека `fuzzywuzzy` использует алгоритм Левенштейна. Метод `partial_ratio` ищет ключевое слово (например, `"time"`) внутри длинной фразы (например, `"what is the time"`). Если находит — возвращает высокий процент совпадения.

### Блок 7: Исполнитель (`execute_cmd`)

Здесь реализована логика конкретных действий.

- **Кроссплатформенность (`paint`)**: Модуль `platform` определяет систему. В Windows вызывается системный `mspaint`. В Linux через `subprocess` программа пытается запустить `pinta`, проверяя как стандартные пути, так и контейнеры `flatpak`.
    
- **Нормализация текста (`ctime`)**: Синтезатор плохо читает цифры (двоеточия сбивают его с толку). Модуль `num2words` переводит числовые значения времени (15:30) в слова ("fifteen hours and thirty minutes").
    
- **Динамический поиск (`youtube`)**: Скрипт берет фразу, удаляет из нее само слово-команду ("youtube" или "video"), а оставшийся "хвост" склеивает с веб-ссылкой через `urllib.parse`, отправляя запрос прямо в браузер.
